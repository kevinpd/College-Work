Scripting languages are programming languages that is a “run-time” environment that will execute the tasks one by one. JavaScript is a scripting language that is very useful for a website as it allows dynamic programming and makes the website interactive. Scripting language mainly uses a batch file therefore using an external file. JavaScript can be important for the user when they are trying to secure code or information that they do not want to be taken. When it comes to security on the server-side it is traditionally meaning so that the programmer puts on as much code onto the server as they can. This means that the server will be doing the most of the protection and heavy lifting. However, it is important to think about the purpose of what the programmer is trying to build for example if the programmer is trying to build an application that works offline they wouldn’t be able to utilise server-side hosting because server-side hosting requires the end-user to have internet connection. There will never be a 100% protected code however for many programmers they now use obfuscation which is the method of changing all of the program code so that it becomes impossible to read and understand at first glance. However with this being said it must still keep its functionality, this method takes time and the understanding of heavy and complex code. Like everything there are also disadvantages to this.

 

Object orientated programs are lines of code within an object that gives the object its “personality”. Object orientated allows the user to give both data and functions to the object which allows the object to then follow those instructions. These instructions can also be connected with other objects which creates relationships between objects allowing the tasks to be completed quicker. An example could be that objects are able to inherit/take other characteristics from other objects. Scripting languages can be used to perform difficult algorithms and calculations that would be very difficult for a human. These calculations would also be changing as they would be set as variables allowing each end-user to have different experiences and outcomes. For example on the webpage load it may show the date. Another example would be using this in things like Amazon where the program has to calculate the price and quantity of multiple different items and then displaying the total. Scripting languages are also useful for reading and hiding information until the user may interact with the trigger. This allows the program to have it loaded before the user presses the trigger allowing a less load time. For example a webpage may hide information about something until the trigger is activated, and then show the information.

 

Event driven programming is a method that allows the programmer to set instructions for the program to follow once it has been triggered. This could be lines of code that will never be triggered therefore never being run. The program will run depending on the events that have been set by the programmer. There are different events that can be set to the program depending on what may be available to the end-user. For example the programmer can set events like a key press, mouse click or even sensors. This is used for user interactivity and can be vital to web applications, where the user will input and a graphical output will occur. With event driven programming there is usually a main loop that is set which waits for a trigger and then plays the call back function when the event is detected from the user. An example of an event-driven program could be as simple as a button which the user can click and then an alert box or dialogue box will show up, providing the user with information.

 

To enhance security, the programmer is able to create a login and password. The method of creating a login and password is actually a simple concept. The programmer will create an external database file where all the new/current user’s information will be kept for the user to access their information. When a new user has been signed up it will be added to the database and then allow the user to login using the information given to sign in. The way the code will work is by checking the information that the end-user has inputted with all the records on the external database file. Usually the external file is kept on the server so that the user is able to access it only when they have an internet connection. This keeps a user’s information secure from anyone that is not meant to have the information. Usually information is encrypted about the user so that not even the owner of the website can access their information keeping it secure from prying eyes.

Redirecting the user from one webpage to another can be useful for multiple reasons. One of them is because the user may have been given or clicked on the business’s old URL as they may have moved the website to a new URL and instead of being given the error “404 Not Found” they would be redirected to the new website without any hassle. For example, if an online retail business changed URL overnight and their old URL was no longer in use they would then be able to use that URL to redirect their users to the new website. The old URL could also contain information that they may want to be notified about and would have a timer so that it doesn’t redirect immediately. A simple redirecting code is with the function “redirect()” with the line “window.location.href “example.com”;” which will redirect the user to the website.

Cookies are information that are stored on the end-user’s computer as text files. Cookies are just information about the end-user that the web-server may want to know, to make their experience better. Creating cookies are quite simple, the code for this is “document.cookie = ‘username=example two’”. When creating a cookie it is important to remember that cookies are created in name-value pairs, as shown from the example. Cookies can be set to delete within a certain amount of time or date. Cookies can also be set to a certain path, cookies will default to the current page if not mentioned. Cookies can be read by simply setting the cookie as a variable which will allow the webpage to use that variable and that being represented as the cookie. Cookies help improve functionality because this means that the user does not need to login or sign up to have their information saved. Cookies can be turned off for the webpage however usually users allow the webpage to save cookies so that the next time they may visit the webpage it will load or have information tailored to them.
