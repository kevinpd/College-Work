Cascading Style Sheets (from here on known as CSS) is a coding language that is built for making the website look a certain way or to style the website. CSS3 is the newest version of CSS and with it comes a lot more from CSS2.1 allowing the user to design their websites much easier. An analogy that I like to use to describe CSS is that it is the skin to the bone, so HTML is considered the skeletal language for website building then CSS can be considered as the skin or makeup for the website allowing the user to make the website however they see fit. CSS has different ways in which it can be linked into the HTML code. There are three which I will be speaking of; inline, embedded and external styles, all used to link HTML with CSS.

With inline style it allows the user to put the CSS code straight into the element. This allows the user to put the code straight into the element without having to go back into the CSS file or going up to the head to change or add code up there. Inline will overwrite all the other code from any other place. For example if a text tag was coloured to black in an embedded/internal however the same text tag was then coloured blue within the element, the outcome of the text would be blue. An example of CSS inline can be shown from the example shown.

Another style that CSS can be linked in HTML is embedded/internal. This is when the user will setup the code by using the tag “<style type=”text/css”>” which will tell the HTML file that until the end of this tag there will be CSS coding between it. This allows the user to add CSS into the HTML in the exact same format as the CSS file however just in the same file. With Internal/ Embedded style sheets it allows the user to overwrite any code from the external stylesheet however will be overwritten by inline stylesheets.

The last style is the external stylesheets. This is a stylesheet that is commonly used when making websites. Like the internal/embedded stylesheets this will also be done in the head tag, however this will not be where the code is placed instead this is a reference link to a file where all of the CSS code is placed. If anything goes wrong with the CSS file or you want to start over with the colour and format but want the content, the user is able to change the file name to the one they would like. This makes it simple to change the site if a new event comes on or adding a temporary scheme to the website. External stylesheets is at the back of the board meaning that every other stylesheets will allow the user to overwrite the external stylesheet.

When using CSS there are different selectors that need to be used in order to reference what code will be applied to which element. The first selector I’ll be showing you, type, which is a selector that allows the user to select a HTML tag. This is an easy method for allowing users to put in a default text font or to give a specific theme to the website. CSS type selects the HTML tag by inputting the element tag. This will select all of the elements with that tag, making it easy to create themes for the website. Both CSS classes and ID will overwrite the code written in the CSS type. The example shows all paragraphs being selected using the CSS type and the code after making all the selected paragraphs into Arial font with a font colour of red and aligned to the center.

Another selector that can be used is the class selector. To select a class CSS uses a period (a dot “.”) which will then allow CSS to search through all the classes and pick the referenced class. Classes are used to separate the ones that are wanted and ones that don’t. For example if the user wanted only three paragraphs and not all the “p” tags then they would likely use a class and then be named something appropriate. The best way to describe classes are just as they sound, in a class there are many students (elements). So in this example it uses the class “.intro” which was created in the HTML file. To create a class the user must do this in the element tag with the code “class= “intro”” which will then allow CSS to find this as a class.

When creating a website, it is important to keep in mind the box model method. This method is usually a term to describe a box that goes around every element and then the user is able to position everything without problems. The box model will consist of margins, borders, padding and then the HTML element. The box model method uses div tags which allows all the HTML elements to be grouped or put together and then allow the user to move around and position everything the way they would like. To position everything, the user will need to use the margin tag. This tag is important in getting the correct position for the div tags. It is important to set the position of the div tag, for example the position I have used is “absolute” which will set to be exactly where I’ve put it no matter the other content. The example shows margin being used from only the top as the code is structured like this “margin-top:100px;” which sets it 100 pixels away from the top of the page. When the user has not set where the margin is from it will default to top and left being equal. Height and width is very important when thinking about the div tag, because if the container is too small then the content may be cut off or may be shuffled in positions that the user may not want. Padding is also very important because this will determine the spacing between the container and the content of the container. In this example I have used a padding of 25 pixels because this is enough to make sure that the content is not pushed against the container. The border of the container is by personal preference and what the user needs. Sometimes the border may not be needed and then the user will set the “border-style” to none. However, in this example I have used a solid border that is green and this will be 5 pixels wide.

In website creation there are multiple different uses when it comes to CSS. A use for CSS could be as simple as changing the background colour. The user is able to change the background colour by using the code as shown in the picture. This code uses a selector to reference the code to be used for that element. For example, the screenshot shows code where it will reference the code to be for the element “body” then add the code between which changes the colour of the background to being a light blue. The structure of the code for this is usually shown by “background-color:white;”. CSS recognises most of colours by word however many users will use a hex code to be more precise or even using RGB values.

CSS also allows the user to add images as a background or in containers. In the screenshot shown it changes the background to an image. The image can be referenced to a file in a folder location or an online image. However not all images work as they must contain a supported file format and not be a web link. For example, the screenshot shown is an online image that has a file format within the URL. The image that I have used has “.jpg” at the end of it defining the web link as a jpeg image which CSS supports. The structure of this code is usually shown by the format “background-image:url(“example.jpg”);”.

CSS can be used to format any text that the user may have on their webpages. There multiple things that can be done to format the text to suit the user’s needs, from things like the text align to the spacing between characters. I will be showing just a few of the common codes which are used to format text. The one that may be used most commonly is to change the font as the default one may not suit the user’s needs. This line of code, “font-family:Arial;”, is changing the font style to Arial. CSS supports most of the common font styles however not all of them. The user may also want to change the font size, which is a simple line of code which is shown in the screenshot. This line of code “font-size:40px;” will change the size of the heading to 40 pixels large. The last line of code that I will be going through is changing the colour of the text. Just like the others this is very simple line of code which consists of either the colour name which are not all supported, a hex code or a RGB value. The structure of the code looks very similar to the background colour change “color:rgb(255,255,255);” . This line of code will change the colour to white using the RGB value.

In CSS the user is able to apply borders and padding to things like the div tag. When it comes to the border in CSS there are quite a few customisations that allows the use to style it to their need. The tag that is used to create the border is “border-style:dotted;” which will change the border to a dotted line instead of a solid colour. There are many different styles that could be used like the dashed, double, groove, ridge etc. This “border-style” tag allows the user to have values from one to four which represent the top, right, left and bottom border lines. With this tag the user is then able to set a width for the border, so in the example I have used the width is 2 pixels and the code structure would look like this “border-width:2px;”. Another tag that can be used as well as this is the padding. The padding is a tag that is used to create space between elements. For example, if the user were to use padding in a div tag for text it would create space inside the div tag to create the space. Padding can be specified for one place for example the tag “padding-top:50px;” would apply a padding only to the top of the paragraph. When the user becomes familiar with this they could use the shorthand method which looks like the example, “padding: 50px 30px 50px 80px;”, which does the same however has less writing.

The heading styles are tags that allow the user to create structure for the website. Heading styles is by ordering the heading tags. The main heading being “<h1>” and the higher the number the lower the heading is. For example a “<h1>” tag would be the main heading and a “<h2>” would be the sub heading below this. This allows structure in the website and these heading styles also have a default font and size, however can be changed. In the example, there is the use of both <h1> and <h2> there have been no alterations to the styles besides the colour. The main heading has a default size of being larger than the sub heading allowing the user to set out the page without the hassle of changing each one individually.
